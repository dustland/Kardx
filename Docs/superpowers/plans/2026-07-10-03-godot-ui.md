# OpenCards Godot UI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build the complete Deck Builder → Mulligan → Match → Result user flow around the tested rules and AI.

**Architecture:** `scripts/main.gd` is a thin screen router. Focused scenes render copied snapshots and translate clicks or drags into `GameAction` objects; they never mutate state. Shared card and zone components are reused by deck construction, mulligan, hand, Support Lines, and Frontline.

**Tech Stack:** Godot 4.7 Controls and scenes, typed GDScript, rules/content/AI from Plans 01–02, headless UI smoke tests.

## Global Constraints

- Complete Plans 01 and 02 with all tests passing before this plan.
- The first screen is Deck Builder or deck selection, never the standalone card list.
- Approved visual direction is Tactical Command Table: military green, brass accents, restrained borders, clear tactical symbols.
- Use `Credit` everywhere.
- Player and AI submit the same `GameAction` interface.
- Click and drag interactions must resolve to the same action builder functions.
- Primary viewport is 1280x720; narrow desktop layouts must not overlap or hide required commands.
- All user-visible card frames, text, costs, and stats are rendered by Godot, not baked into images.

- [ ] **Step 3: Implement a thin router**

```gdscript
# scripts/main.gd
class_name Main
extends Control

const VALID_SCREENS := ["deck_builder", "mulligan", "match", "result"]
const SCREEN_PATHS := {
    "deck_builder": "res://scenes/ui/deck_builder_view.tscn",
    "mulligan": "res://scenes/ui/mulligan_view.tscn",
    "match": "res://scenes/ui/match_view.tscn",
    "result": "res://scenes/ui/result_view.tscn",
}

@onready var screen_host: Control = %ScreenHost
var current_screen: Control
var catalog: ContentCatalog
var controller: MatchController
var ai: AIPlayer

func show_screen(screen_name: String, payload: Dictionary = {}) -> void:
    assert(screen_name in VALID_SCREENS)
    if current_screen:
        current_screen.queue_free()
    current_screen = load(SCREEN_PATHS[screen_name]).instantiate()
    screen_host.add_child(current_screen)
    current_screen.initialize(self, payload)
```

`main.tscn` contains a full-rect `Control`, background `ColorRect`, and full-rect `ScreenHost`. `_ready()` loads and validates content, applies the generated Theme, selects `us-starter` when no local preference exists, and opens Deck Builder.

If startup validation returns diagnostics, instantiate `content_error_view.tscn` instead of a normal screen. It shows the sorted `code`, `path`, and `message` fields in a scrollable panel and offers only Quit and Retry Validation; Deck Builder and Match remain inaccessible until validation succeeds.

`card_view.tscn` uses a `Button` root with fixed mode sizes: catalog `180x252`, hand `116x162`, battlefield `84x118`.
```gdscript
func bind(data: Dictionary, display_mode: String) -> void:
    card_data = data.duplicate(true)
    mode = display_mode
    %Title.text = data.get("title", "") if not data.get("hidden", false) else ""
    %Artwork.texture = _load_art(data.get("image_path", FALLBACK_ART))
    %Attack.text = str(data.get("attack", ""))
    %Defense.text = str(data.get("defense", ""))
    tooltip_text = data.get("description", "")
    _apply_mode_size()
```

Use `_get_drag_data()` and `_can_drop_data()` for drag interaction; pressing emits the same instance ID so views can call the same `ActionBuilder` methods.
